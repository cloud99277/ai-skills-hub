# CLI 工具踩坑预警库

> **适用**：CLI 工具 / 脚本型项目
> **维护方式**：CLI 项目经验沉淀时追加

---

## 🟡 P0: 参数解析异常时必须有友好提示

```
场景：用户传错参数时，程序报不可理解的错误
修复：使用参数解析库（argparse/click/commander）+ 自定义 help 信息
预防：定义 --help 输出和常见错误的友好提示
```

## 🟡 P1: 文件路径兼容性

```
场景：在 Windows/Mac/Linux 之间路径分隔符不同
修复：使用 os.path.join() 或 pathlib 而非硬编码路径分隔符
预防：开发时意识到跨平台兼容
```

## 🟡 P2: 大文件/大量文件处理的内存控制

```
场景：一次性读取大文件导致内存溢出
修复：使用流式读取（readline / streaming）
预防：处理文件时默认使用流式 API
```

## 🟢 P3: 退出码规范

```
场景：脚本失败但返回退出码 0，上游调用者以为成功
修复：成功返回 0，失败返回非零（按错误类型分不同退出码）
预防：在 catch 块中 sys.exit(1)
```

## 🟡 P4: 自写解析器必须用真实数据调试

```
场景：自写的 YAML 子集解析器对简单测试通过，但碰到真实的 2 层嵌套 YAML（如 input: 下有子属性，output: 是兄弟属性）时分不清父子与兄弟关系
根因：解析器只用缩进级别推断层级，但没记录「子字典起始行的缩进」来判断归属
修复：跟踪 current_sub_indent，当新行缩进 > sub_indent 才算子属性，否则算兄弟
预防：自写解析器必须用真实的 YAML 数据（非最简 case）做冒烟测试
来源：agent-os Phase 4 YAML 子集解析器 bug
```
