# 调研自检清单（Quality Gate）

> Quick Mode 用此清单快速自检。Full Mode 自检后再调用 `project-audit`。

## 选品检查

- [ ] 包含 ≥2 个同量级/同定位方案？
- [ ] 包含 ≥1 个反面教材（失败/停滞/放弃）？
- [ ] 总项目数 5-8 个？
- [ ] 不是全部"重量级/知名"项目？

## 分析检查

- [ ] 每个项目有架构描述（不只一句话介绍）？
- [ ] 对比矩阵有量化评分（0-3 分制）？
- [ ] 反面教材有"失败根因分析"（不只是说"停滞了"）？

## 借鉴检查

- [ ] 每条借鉴标注了锚点可实现度（几成）？
- [ ] 每条借鉴标注了决策类型（Adopt/Adapt/Extract/Anti-Pattern/Skip）？
- [ ] 有"被砍清单"（引力陷阱过滤掉的 + 原因）？
- [ ] 借鉴项 ≤ 10 条？

## 定位检查

- [ ] "独特定位"同时列出了优势和代价？
- [ ] 代价分析是诚实的（不是在辩护）？

## 格式检查

- [ ] 有对比矩阵总览？
- [ ] 有评估维度说明？
- [ ] 数据来源有 URL 或引用？
- [ ] 末尾有 YAML 机器可读摘要（`research_summary`）？

## 质量标准（继承自 market-research）

1. 每个重要结论需要来源（URL/证据）
2. 优先近 12 个月的信息，标注过时数据
3. 包含反面观点和失败案例
4. 调研服务于决策，不做"调研表演"
5. 区分事实、推断和建议
6. 包含同量级方案（避免选品偏差）
7. 每条借鉴有"现有工具能做到几成"评估
8. 独特定位必须配代价分析
